# 端口被占用的解决方法

## 1. 问题现象

运行 `canteen` 项目的 Spring Boot 后端时，控制台出现类似报错：

```text
APPLICATION FAILED TO START

Description:

Web server failed to start. Port 9090 was already in use

Action:

Identify and stop the process that's listening on port 9090 or configure this application to listen on another port.
```

该报错表示：后端服务启动时需要使用 `9090` 端口，但该端口已经被其他进程占用，所以 Spring Boot 无法启动。

## 2. 本项目为什么会使用 9090 端口

后端端口配置位于：

```text
springboot/src/main/resources/application.yml
```

配置内容为：

```yaml
server:
  port: ${SERVER_PORT:9090}
```

含义是：

- 如果系统环境变量中设置了 `SERVER_PORT`，则使用该环境变量的值。
- 如果没有设置 `SERVER_PORT`，则默认使用 `9090`。

因此，本项目默认启动端口是 `9090`。

## 3. 查看 9090 端口是否被占用

在 PowerShell 中执行：

```powershell
netstat -ano | findstr ":9090"
```

如果没有任何输出，说明 `9090` 当前空闲，可以正常启动后端。

如果出现类似内容：

```text
TCP    0.0.0.0:9090    0.0.0.0:0    LISTENING    29272
TCP    [::]:9090       [::]:0       LISTENING    29272
```

说明 `9090` 已经被占用。

最后一列数字是进程 PID，例如上面的 `29272`。

## 4. 查看占用端口的进程

假设查到的 PID 是 `29272`，可以继续执行：

```powershell
Get-Process -Id 29272
```

如果结果显示进程名为 `java`，通常说明当前已经有一个 Java 程序正在运行。

对于本项目而言，很可能是之前启动过的 Spring Boot 后端还没有关闭。

## 5. 结束占用端口的进程

如果确认该进程就是之前残留的后端服务，可以执行：

```powershell
taskkill /PID 29272 /F
```

其中：

- `29272` 需要替换为实际查到的 PID。
- `/F` 表示强制结束该进程。

结束后，再次检查端口：

```powershell
netstat -ano | findstr ":9090"
```

如果没有输出，说明端口已经释放。

## 6. 再次启动项目

端口释放后，重新启动 Spring Boot 后端即可。

常见启动方式：

```powershell
cd springboot
mvn spring-boot:run
```

也可以通过 IDE 运行：

```text
SpringbootApplication.java
```

启动成功后，控制台一般会显示 Tomcat 已经在 `9090` 端口启动。

## 7. 如何避免以后重复出现

建议以后运行毕设项目时注意以下几点：

1. 不要重复启动后端。
   如果 Spring Boot 已经启动成功，就不要再次点击运行按钮。

2. 每次启动失败时，先检查 `9090` 是否被占用。

   ```powershell
   netstat -ano | findstr ":9090"
   ```

3. 如果端口被 `java` 占用，优先判断是不是自己之前启动的后端。

4. 如果要重新运行项目，先停止旧的后端进程，再启动新的后端。

5. 检查 IDE 的运行配置。
   如果日志中出现了与当前项目不一致的路径，说明可能运行到了旧项目或旧编译目录，需要调整运行配置。

6. 不建议随意修改后端端口。
   因为本项目的 Web 管理端和微信小程序端也默认连接 `9090`，随意改端口可能导致前端无法访问后端。

## 8. 快速处理命令

如果以后再次遇到 `Port 9090 was already in use`，可以按下面流程处理：

```powershell
# 1. 查询 9090 端口占用情况
netstat -ano | findstr ":9090"

# 2. 假设查到 PID 是 29272，查看进程
Get-Process -Id 29272

# 3. 确认可以结束后，释放端口
taskkill /PID 29272 /F

# 4. 再次确认端口是否空闲
netstat -ano | findstr ":9090"
```

当第 4 步没有任何输出时，说明 `9090` 已经空闲，可以重新启动后端。
